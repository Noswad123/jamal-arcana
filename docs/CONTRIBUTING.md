# Contributing

Jamal Arcana is currently a personal ecosystem becoming public. Contributions should preserve the core shape: small independent tools with clear composition boundaries.

## Documentation contributions

Good docs changes include:

- clearer installation notes
- better examples
- corrections to package links or release status
- composition recipes
- lore that helps explain a tool without obscuring it

## Package contributions

Package source lives in the package repositories:

- <https://github.com/Noswad123/wisp>
- <https://github.com/Noswad123/waystone>

Open implementation issues or pull requests in those repositories rather than this umbrella docs repo.

## Tool criteria

Before adding a new tool to the catalog, ask:

1. Does it work independently?
2. Does it compose through the shell?
3. Does it avoid dotfiles-only assumptions?
4. Is the name memorable and behaviorally accurate?
5. Is there a public repository or clear path to one?
