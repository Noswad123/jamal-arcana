# Vision

Jamal Arcana is a collection of small, local-first tools that make a computer feel more personal, navigable, and enchanted.

The ecosystem grew out of dotfiles, but it should not require anyone to adopt those dotfiles. Each tool should stand alone, be easy to install, and compose naturally with the shell.

## What belongs in Jamal Arcana

A tool belongs here when it is:

- **local-first** — useful on a personal machine without a hosted service
- **small and sharp** — focused on one clear workflow
- **composable** — works with other tools through ordinary commands, files, and pipes
- **tasteful** — carries a bit of lore, naming, and craft
- **portable enough** — not tied to one private dotfiles layout

## What does not belong

Jamal Arcana is not:

- a monolithic productivity platform
- a replacement for dotfiles
- a SaaS suite
- a package source repository for every tool

This repository is the map. Individual package repositories are the workshops.

## Principles

1. **Modular by default** — every tool should work on its own.
2. **Composable by convention** — integration should happen through the shell.
3. **No dotfiles dependency** — personal config can enhance tools, but not be required.
4. **XDG-friendly state** — runtime state should live under standard user paths.
5. **Magic as interface** — the naming should make tools memorable without hiding what they do.
